Zod Integration for Validation

This section will demonstrate how to use Zod for schema validation in Angular forms.

Setting up Zod

Zod is a TypeScript-first schema declaration and validation library. To use it, you first need to install it:

npm install zod

After installation, you can import it into your TypeScript files:

import * as z from 'zod'; // Or: import { z } from 'zod';

Creating a Basic Zod Schema

You define a schema by chaining Zod's validation methods.


const UserSchema = z.object({{'{'}}
  username: z.string().min(3, {{'{'}} message: "Username must be at least 3 characters long" }),
  email: z.string().email({{'{'}} message: "Invalid email address" }),
  age: z.number().min(18, {{'{'}} message: "Must be 18 or older" }).optional(),
  isAdmin: z.boolean().default(false)
});

// You can infer the TypeScript type from the schema:
// type User = z.infer;

Formly Form with Zod Validation (Submit-time)

This Formly form's fields are generated based on the same Zod schema (UserSchema). Validation using Zod is performed when the form is submitted.

Validation errors for this Formly example are shown in the console and via an alert on submit. Integrating Zod errors directly into Formly's per-field error messages requires more advanced setup (e.g., custom Formly validators, wrappers, or extensions that can bridge Zod's error object with Formly's field state). The mapZodToFormlyFields utility in this component provides a basic mapping of Zod schema properties (like 'required' or 'minLength') and some error messages to Formly's field configuration.
